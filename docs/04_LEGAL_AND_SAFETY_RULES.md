# 04 — Legal & Safety Rules
## Quantum Mentor World | Quantum Mentor Official

---

## Overview

Quantum Mentor World is a strictly **legal, safe, and ethical** educational resource directory. The platform's core principle is that every resource listed must be verified, approved, and safe for users to access.

This document defines the complete legal and safety policy for all content, resources, links, and media on the platform.

---

## Platform Legal Foundation

Quantum Mentor World operates as a **resource discovery and directory platform**, not as a hosting or distribution platform. The website:

- Curates and indexes external legal resources
- Provides verified links to official and legal sources
- Does **not** host or distribute copyrighted content without authorization
- Does **not** scrape, copy, or republish full copyrighted articles without permission
- Provides original editorial descriptions and summaries
- Properly attributes all sources

---

The following content types are approved and allowed:

| Approved Content | Description |
|---|---|

| **Open-Source Resources** | Resources released under recognized open-source licenses (MIT, GPL, Apache, etc.) |

| **Open-Access Books** | Books published under open-access licenses |
| **Educational Tutorials** | Publicly available tutorial content from verified sources |
| **GitHub Open-Source Repositories** | Public repositories with recognized licenses |

## Section 3: Legal Status Classification

Every resource on the platform must have one of the following legal statuses:

| Status | Meaning |
|---|---|
| `Approved` | Resource has been verified and confirmed legal |
| `Pending Review` | Resource has been added but not yet verified |
| `Under Review` | Resource is currently being reviewed for legal compliance |
| `Rejected` | Resource failed legal review and is removed from public view |
| `Requires Attribution` | Resource is legal but requires proper credit display |

---

## Section 4: Safety Status Classification

Every resource must also have a safety status:

| Status | Meaning |
|---|---|
| `Safe` | Resource has been verified safe, links are to official sources |
| `Caution` | Resource is legal but users should exercise some caution |
| `Unverified` | Resource has not yet been safety-checked (hidden from public) |

---

## Section 5: License Type Classification

Resources must clearly display their license type:

| License | Description |
|---|---|
| `MIT` | Open-source, permissive, allows commercial use |
| `GPL-2.0` | Open-source, copyleft, requires derivatives to also be GPL |
| `GPL-3.0` | Updated GPL version with additional protections |
| `Apache 2.0` | Open-source, permissive with patent protection |
| `BSD 2-Clause` | Open-source, permissive |
| `BSD 3-Clause` | Open-source, permissive with non-endorsement clause |
| `CC0 (Public Domain)` | No rights reserved, completely free |
| `CC BY` | Free to use with attribution |
| `CC BY-SA` | Free with attribution, derivatives must use same license |
| `CC BY-NC` | Free for non-commercial use only |
| `Freeware` | Free to use, not open-source |
| `Freemium` | Free tier available, paid tier exists |
| `Open Access` | Freely available for reading, may have restrictions on redistribution |
| `Public Domain` | Copyright expired, free for all use |
| `Proprietary` | Not open-source; only listed if creator approved and free |

## Section 6: Content Review Process

### For New Resources Added by Admin

1. **Admin adds resource** → Status set to `Pending Review`
2. **Admin reviews resource** using the Legal Review panel
   - Confirms the official source
   - Verifies the license type
   - Checks the download/embed URL is from an official source
   - Confirms no malware or phishing indicators
3. **Admin approves resource** → Status updated to `Approved` → Resource goes live
4. **Admin rejects resource** → Status set to `Rejected` → Resource stays hidden

### For External Link Safety

- All external links must point to official, recognized domains
- Links are visually marked as "External Link" with a clear icon
- An exit notice is displayed when users follow external links
- Links are periodically reviewed to ensure they remain valid and safe
- Dead or redirected links are flagged for review

---

## Section 7: Copyright & Attribution Rules

### For News Articles
- Full article text must NOT be copied from external sources
- Only original summaries, excerpts (under fair use), and commentary are allowed
- All news items must include:
  - Original source name
  - Original source URL
  - Author name (if available)
  - Published date

### For Embedded Videos
- Only embed code from official platforms (YouTube, Vimeo, Archive.org) may be used
- Embed source and original creator must be credited
- No downloading or rehosting of video content

### For Books
- Only books with open-access licenses, public-domain status, or creator approval may be listed
- Download links must point to official or authorized sources only

## Section 8: User Responsibility Notice

The website will display the following notice on resource pages:

> **Disclaimer:** Quantum Mentor World is a resource directory platform. All external links lead to third-party websites. Quantum Mentor World does not host or distribute the linked content and cannot guarantee the accuracy or safety of external websites. Users access external resources at their own risk. Always verify the official source before downloading or using any resource.

---

## Section 9: Reporting Illegal or Unsafe Content

A content reporting mechanism will be implemented to allow users to flag potentially illegal or unsafe content:

- Users can click "Report This Resource" on any resource page
- Report goes to the admin contact messages
- Admin reviews the report within a reasonable time
- Confirmed illegal/unsafe resources are immediately unpublished

---

## Section 10: Admin Accountability

- All admin actions are logged in the `admin_activity_logs` table
- Logs include: which admin performed the action, what resource was affected, timestamp
- This ensures accountability and auditability of all content decisions
